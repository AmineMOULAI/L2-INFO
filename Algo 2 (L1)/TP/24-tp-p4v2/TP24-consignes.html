Consignes

En quels formats, le sujet est disponible ?

Sur ce jupyterbook, les sujets sont disponibles en version :

  • .ipynb: le notebook jupyter "d'épreuve" à compléter et, le cas échéant, à déposer en fin d'épreuve
  • html

Comment utiliser les notebooks d'épreuve ?

  • On rappelle les consignes de manipulation de ces notebooks d'épreuve.
  • Bien lire TOUT ce qui suit et AU MOINS DEUX FOIS.

Quoi répondre ?

  • Les questions demandent des réponses sous forme de code python (la plupart) ou sous forme de texte.
  • Sauf mention explicite (questions de fin de TP), aucune cellule ne demande des entrées-clavier avec input().
    • Veillez bien à effacer ces instructions si vous en avez ajouté pour tester vos réponses bien que ce ne soit pas nécessaire : voir Sur les tests de vérification un peu plus loin.

Où répondre ?

  • Uniquement dans les cellules prévues à cet effet :
    • chaque question est suivie d'une cellule pour cela (ou plus rarement plusieurs).
  • Les autres cellules ne sont pas modifiables :

    # Ne pas écrire dans cette cellule

  • Ne surtout pas ajouter de cellule dans ce notebook
    • ni faire de copier-coller d'une cellule à la place d'une autre (chaque cellule a un numéro caché qui lui est unique)
    • en revanche, vous pouvez copier le contenu d'une cellule et le coller dans une autre.

Comment répondre avec du code python ?

Les cellules dédiées à vos réponses avec du code python se reconnaissent par ce message :

# ENTRER VOTRE CODE A LA PLACE DE CES 2 LIGNES
raise NotImplementedError()

  • Vous pouvez supprimer la première de ces 2 lignes ou la laisser -- c'est un commentaire.
  • Votre réponse prendra la place de la seconde ligne raise NotImplementedError()
  • Si vous n'avez pas de réponse à donner, 2 possibilités :
    1. Si vous ne modifiez pas cette cellule, l'exception NotImplementedError() bloquera l'exécution de la cellule et donc du reste du notebook à partir de là.
    2. Si vous effacez cette ligne raise NotImplementedError(), vous pouvez continuer à répondre à la suite du sujet et aussi relancer votre noyau pour exécuter des cellules situées après celle-ci.

Comment répondre avec du texte ?

Les cellules dédiées à vos réponses textuelles, en markdown par exemple, se reconnaissent par le message :

VOTRE REPONSE A LA PLACE DE CE TEXTE (format texte ou markdown)

Sur les fonctions demandées.

  • Respectez l'en-tête fournie pour chaque fonction demandée.
  • Elle est utilisée pour les vérifications : celles qui vous aident et aussi celles qui accompagnent automatiquement l'évaluation de votre travail.

Sur les tests de vérification.

  • Un nombre important de questions est suivi de tests pour vérifier la validité de votre solution.
    • soit avec des questions spécifiques ou sous forme d'une cellule dédiée.
  • Ces cellules de tests prennent une des deux formes suivantes.
    1. cellule à compléter selon des indications et un exemple de résultat attendu.
    2. cellule non modifiable, à exécuter directement et qui se termine par une ou plusieurs instructions assert.
    • Un test qui déclenche une AssertionError() prouve que votre traitement n'est pas correct.
    • Il est fortement conseillé de ne pas continuer aveuglément la résolution d'une section du sujet si un test n'est pas satisfait.
      • Cependant, si votre traitement ne vérifie pas ce test et que vous souhaitez continuer avec un notebook qui s'exécute sans erreur, commentez ces instructions assert.

Vérifications à effectuer à la fin de votre travail.

Assurez-vous que tout fonctionne comme prévu. C-a-d :

  • Redémarrer le noyau en mode →\rightarrow→Redémarrer & tout exécuter (ou Restart and run all)
  • Votre notebook doit s'exécuter entièrement comme prévu et sans erreur nouvelle !
  • L'exécution d'un notebook avec des cellules sans réponse et non modifiées peut être interrompue par l'exception NotImplementedError() ou AssertionError() (relire plus haut).

Les 2 derniers conseils 😃

  • Enregistrez régulièrement votre notebook.
  • Redémarrer régulièrement le noyau en exécutant toutes les cellules.
  • Relisez ces consignes.
